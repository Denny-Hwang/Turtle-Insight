# PRD — Turtle Insight

제품 요구사항 문서. v0.1.

## 1. 목적

세상의 메가트렌드를 진단하고, 그 근거 위에서 산업·기업·기술을 전망해, 사용자(소유자) 한 명의 투자 방향을 *제안*하고 정기적으로 갱신한다. 분석은 전 세계 무경계, 실행은 한국·미국 시장 우선.

## 2. 사용자 & 문제

- **사용자**: 단독 소유자(개발자 본인). 장기(5–20년) 호흡 + 단기(1–5년) 전술을 병행하는 개인 투자자.
- **문제**: (a) 메가트렌드→산업→기업의 인과를 일관된 틀로 추적하기 어렵다. (b) 분석이 단발성이라 갱신·검증·반증이 안 된다. (c) 확신과 실제 적중을 구분(캘리브레이션)하지 못한다. (d) 정보가 미·한·글로벌에 흩어져 있다.

## 3. 가치 제안

3계층 **테제 그래프**로 철학→트렌드→기업을 인과로 잇고, 각 테제에 근거·반증조건·확신도를 붙여 *지속 검증*하며, 레드팀과 트랙레코드로 *규율 있는* 판단을 돕는다.

## 4. 범위

### In scope
- 메가트렌드/산업/기업 테제의 생성·검증·반박·갱신 (테제 그래프)
- KR/US 우선의 데이터 수집(공시·매크로·시세·뉴스 링크·키노트)
- 위험성향·시계 등 사용자 제약 반영한 워치리스트/배분 **제안**
- 데일리/위클리/먼슬리/온디맨드 브리핑
- 캘리브레이션 트랙레코드(예측 대비 실현 채점)

### Out of scope (non-goals)
- **자동 매매·주문 실행** (영구 제외)
- 타인 대상 투자자문/배포·유료 서비스 (개인용)
- 고빈도 트레이딩/실시간 체결 신호
- 뉴스 전문 저장·재배포
- 보장된 수익률·확정적 매수 지시

## 5. 핵심 사용 시나리오

1. **트렌드 진단**: "AI가 에이전트 시스템으로 가면 어디에 수요가 생기나?" → Strategist가 Chain 테제(메모리·전력 등) 생성, 근거·반증조건 제시.
2. **기업 검증**: 특정 기업이 어떤 테제의 수혜인지, 펀더멘털이 테제를 지지/반증하는지 Analyst가 판정.
3. **반론 확인**: 내가 확신하는 테제의 베어 케이스와 "무엇이 보이면 틀린 것인가"를 RedTeam이 제시.
4. **방향 제안**: 생존 테제 + 내 위험성향으로 워치리스트/배분 시나리오를 Allocator가 제안.
5. **정기 업데이트**: 매일/매주 브리핑으로 변동·신규/반증 테제·시장국면을 받는다.
6. **자기 점검**: 월간 캘리브레이션 스코어카드로 내 판단의 적중률을 본다.

## 6. 기능 요구사항(FR)

- FR-1 시그널 수집·정규화·태깅(링크+요약+메타만).
- FR-2 테제 CRUD + 상태기계(`draft→candidate→active→invalidated|realized`).
- FR-3 테제 그래프(부모/자식 링크, 자산 링크) 탐색.
- FR-4 스키마 검증: 필수 `falsifiers`·`evidence(url,date)`.
- FR-5 RedTeam 게이트: 미통과 시 `active` 차단.
- FR-6 확신도 입력 + 트랙레코드 채점.
- FR-7 제약(위험성향·시계·제외 섹터) 기반 제안 생성(시나리오·사이징 논리).
- FR-8 브리핑 생성(4종).
- FR-9 데이터 소스 ToS 준수(전문 미저장).
- FR-10 로컬 단독 운용(Mac Mini), 스케줄 구동.

## 7. 비기능 요구사항(NFR)

- 로컬 우선·오프라인 내성(시세/캐시). 비밀정보 env 격리.
- 추론 비용 통제: 고빈도 작업은 로컬/경량 모델, 심층은 Claude(모델 config).
- 재현성: 테제·근거·결정이 git 이력으로 추적(content-as-code).
- 관측성: 에이전트 실행 로그, 캘리브레이션 메트릭.

## 8. MVP 정의 (v1.0)

씨앗 테제 1줄기(지능의 유틸리티화 → 에이전트 시스템 → 가속컴퓨팅 풀스택 병목 → 자산 후보)를 **끝까지** 돌린다: Scout→Analyst→RedTeam→Allocator→Synthesizer + Curator, SQLite 저장, Streamlit 뷰어, KR+US 소규모 워치리스트.

**성공 기준**: (1) 씨앗 테제가 근거·반증조건·확신도를 갖추고 스키마·RedTeam 게이트를 통과해 `active`가 된다. (2) 그 테제로 시나리오가 달린 제안과 위클리 브리핑이 자동 생성된다. (3) 모든 출력이 GOLDEN RULES를 위반하지 않는다(매매 미실행·제안형).

## 9. 리스크

- 데이터 소스 약관/가용성 변동 → 커넥터 추상화로 격리, 링크+요약 원칙.
- 추론 비용 증가 → 모델 티어링·캐시.
- 확증편향 → RedTeam·캘리브레이션을 1급 기능으로.
- 과의존 → 도구는 *결정 지원*, 실행은 사람(설계로 강제).
