# 테제 & 인식론 가이드라인

Turtle Insight의 *판단의 질*을 지키는 규율. 이 문서의 규칙은 코드 게이트·CI·eval로 강제된다. (요약은 `CLAUDE.md §1 GOLDEN RULES`.)

## 1. 테제란 무엇인가

테제(Thesis)는 **검증 가능한 투자 주장**이다. 의견·느낌이 아니라 다음을 항상 동반한다:
- **주장(claim)** — 무엇이, 왜 일어날 것인가.
- **근거(evidence)** — 출처·날짜·링크가 달린 사실. (전문 인용 금지, 짧은 요약만.)
- **반증조건(falsifiers)** — *무엇이 관측되면 이 테제는 틀린 것인가.* 비어 있으면 테제가 아니다.
- **확신도(conviction)** — 0–100, 캘리브레이션 대상.
- **리스크 / 강세·기본·약세 시나리오.**

## 2. 승격 게이트 (절대)

`candidate → active`는 다음을 모두 만족할 때만:
1. `falsifiers`가 1개 이상(구체적·관측 가능).
2. `evidence`가 1개 이상이며 각 항목에 `url`·`date`.
3. RedTeam 리뷰 `verdict == pass`.

미충족 시 active 불가. 코드(`domain/state.py`)와 CI(R1)·eval이 강제.

## 3. 확신도는 *벌어들이는* 값

- 확신도는 근거의 양·질·독립성으로 정당화된다. 근거 없는 고확신 금지.
- 모든 active 테제는 예측을 등록하고, Curator가 실현 여부로 채점(캘리브레이션). 반복적으로 빗나간 출처/추론 패턴은 가중치 하향.
- 오래된 근거(신선도 초과)는 자동 플래그 → 재검토.

## 4. 반론 우선 (RedTeam)

- 모든 테제 변경에 베어 케이스·반증조건 점검·확증편향 플래그를 생성.
- 확증 근거만 모으는 것을 금한다. 가장 *구체적인* 반증 신호를 먼저 확인.
- 사용자가 강하게 믿는 테제일수록 RedTeam을 더 엄격히 적용.

## 5. 분석 ≠ 결정 (산출물 형태)

- 산출물은 **제안**이다: 시나리오(강세/기본/약세) + 확률감 + 포지션 사이징 *논리* + 리스크.
- "지금 사라/팔아라"식 명령형 단정 출력 금지. 점추정 대신 시나리오·불확실성 표면화.
- 최종 결정·실행은 사람이 한다.

## 6. 매매 미실행 (영구 non-goal)

- 시스템은 주문을 내지 않는다. 증권사/거래 API는 **읽기 전용**만.
- 매매 실행 기능은 만들지 않는다. 관련 요청은 거절하고 사람 실행으로 남긴다.

## 7. 출처·저작권

- 뉴스·유료 콘텐츠: 링크 + ≤500자 사실 요약 + 메타만. 전문 저장·재배포 금지.
- 각 근거는 출처·날짜 필수. 불확실하면 근거에 넣지 않는다(날조 금지).
- 소스 약관(ToS) 준수. 약관이 막으면 그 소스는 쓰지 않는다.

## 8. 위치 설정 (disclaimer)

Turtle Insight는 **개인 의사결정 지원 도구**다. 투자자문·권유가 아니며, 수익을 보장하지 않는다. 시장은 불확실하고 모든 테제는 틀릴 수 있다 — 그래서 반증조건·캘리브레이션·반론이 시스템의 1급 기능이다.
